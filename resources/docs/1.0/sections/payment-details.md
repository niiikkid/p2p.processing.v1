<a name="description"></a>
## Описание
В данном разделе представлен список реквизитов по которым совершается платежные сделки.

<a name="proprs-description"></a>
## Описание параметров
- **Никнейм реквизитов** - это название реквизита, нужно для того чтобы трейдеру было проще ориентироваться по реквизитам внутри системы.
- **Платежный метод** - каждый реквизит может быть прикреплен только к одному платежному методу.
- **Метод** - параметр появляющийся только если параметр "Платежный метод" равен СБП. Это дополнительная настройка описывающая какой именно платежный метод используется внутри системы СБП.
- **Тип реквизита** - всего в системе доступно только 3 типа, но платежный метод может быть ограничен до одного типа реквизитов. Номер телефона, Номер счета, Номер банковской карты.
- **Инициалы (имя получателя)** - инициалы владельца реквизита которые увидит клиент при совершении перевода. Например: Пол Атрейдес М.
- **Лимит на объем операций в сутки** - каждый реквизит можно ограничить по тому сколько в сумме успешных сделок будет по нему совершено в сутки. При достижении лимита реквизит не выключается, он просто перестанет попадать в оборот если оставшийся лимит меньше суммы сделки.
- **Активен** - реквизит можно включить и выключить из оборота сделок.

<a name="important"></a>
## Важные моменты
- Только трейдер может добавлять реквизиты, но администратор может просматиривать и редактировать все возможные реквизиты.
